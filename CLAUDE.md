## Project: UAP Map

An interactive public map of declassified US government UAP/UFO sightings, built with Next.js 14 (App Router), TypeScript, Tailwind CSS, and react-leaflet. Hosted on Vercel. Data sourced from the US DoD PURSUE program at war.gov/ufo — 162 files in the first tranche, with rolling releases every few weeks.

## Architecture

- app/ — Next.js App Router pages and layouts
- components/ — React UI components (map, sidebar, filters, timeline slider)
- lib/ — scraper logic, Claude API extraction, utility functions
- data/sightings.json — the master structured dataset, generated by the scraper

## Data Pipeline

The scraper (lib/scraper.ts) fetches documents from war.gov/ufo, extracts text (PDF via pdf-parse or raw HTML), then calls Claude claude-sonnet-4-6 to extract structured fields per sighting: title, date, location_name, lat, lng, country, agency, object_description, sensor_type, witness_count, duration_minutes, summary_one_line, confidence_score. Output is written to data/sightings.json.

## Key Design Decisions

- Dark classified-dashboard aesthetic throughout, monospace font for data fields, CartoDB dark matter map tiles
- Markers are color-coded by agency and have a CSS pulse animation
- Timeline slider filters by year range with animated marker appearance
- Each sighting has a unique shareable URL via ?id= query param (window.history.replaceState)
- "Most anomalous" panel ranks top 5 sightings by confidence_score — this is the viral hook

## Environment Variables

ANTHROPIC_API_KEY is required for the scraper. Never hardcode it. Always read from process.env.

## Current Status

Frontend complete, ready for Vercel deployment. All features implemented: map with clustering, agency filters, year timeline, sidebar detail panel, shareable URLs, Most Anomalous panel, OG meta tags, "New files drop soon" badge.

## Known Issues

35 sightings have no coordinates and are excluded from map display.

## Commands

- npm run dev — start development server
- npx ts-node lib/scraper.ts — run the scraper to regenerate data/sightings.json
- vercel --prod — deploy to production
